# 2026-06-01 Dev Log

This entry records the second round of June 1 changes in the current working tree.

## What Changed

- Reframed the runtime around `agent` instead of “instance” terminology.
- Made the agent creation UI default-filled so it is ready to create and run immediately.
- Changed rendering to use the algorithm-side vertex array view instead of treating the source mesh as the render primitive.

## Current Runtime Model

- `agent_execute` owns the manual agent composer UI.
- A shared agent can bind render and physics roles at once.
- Physics updates the active vertex array; rendering reads the current vertex array and topology arrays.
- The UI defaults to a shared agent preset and starts the work on creation.

## Validation

- `min_vulkan_win32` builds successfully after the latest changes.
- The remaining build output is the existing Eigen `C4819` warning and the unrelated `pwsh.exe` post-build message.

## Terminology

- In the project docs, Chinese `实例` should be read as `agent` unless a file explicitly says otherwise.
- The agent-level algorithm model is: small algorithms stay in `algorithm_library`, while `agent` composes them through ordered bindings, aliases, and container routes.
- The final agent-level compliance descriptor is what gets passed to `algorithm` / `algorithm_mng` for container allocation.
- A helper composer script is still the preferred way to build and rename merged descriptors before wiring them into an agent.
